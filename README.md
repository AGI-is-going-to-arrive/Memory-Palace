## рҹҢҹ д»Җд№ҲжҳҜ Memory Palaceпјҹ

**Memory PalaceпјҲи®°еҝҶе®«ж®ҝпјү** жҳҜдёҖеҘ—дё“дёә AI Agent жү“йҖ зҡ„й•ҝжңҹи®°еҝҶж“ҚдҪңзі»з»ҹгҖӮе®ғдёәеӨ§иҜӯиЁҖжЁЎеһӢжҸҗдҫӣ **жҢҒд№…еҢ–гҖҒеҸҜжЈҖзҙўгҖҒеҸҜе®Ўи®Ў** зҡ„еӨ–йғЁи®°еҝҶиғҪеҠӣвҖ”вҖ”и®©дҪ зҡ„ Agent дёҚеҶҚ"жҜҸж¬ЎеҜ№иҜқйғҪд»Һйӣ¶ејҖе§Ӣ"гҖӮ

йҖҡиҝҮз»ҹдёҖзҡ„ [MCPпјҲжЁЎеһӢдёҠдёӢж–ҮеҚҸи®®пјү](https://modelcontextprotocol.io/) жҺҘеҸЈпјҢMemory Palace еҸҜж— зјқжҺҘе…Ҙдё»жөҒ AI ејҖеҸ‘е·Ҙе…·вҖ”вҖ”**CodexгҖҒClaude CodeгҖҒGemini CLIгҖҒCursorгҖҒAntigravity**вҖ”вҖ”е®һзҺ°и·ЁдјҡиҜқзҹҘиҜҶз§ҜзҙҜдёҺеҚіж—¶еҸ¬еӣһгҖӮ

## вңЁ ж ёеҝғзү№жҖ§

### рҹ”’ еҸҜе®Ўи®ЎеҶҷе…ҘжөҒж°ҙзәҝ

жҜҸдёҖж¬Ўи®°еҝҶеҶҷе…ҘйғҪз»ҸиҝҮдёҘж јжөҒж°ҙзәҝпјҡ**Write Guard йў„жЈҖ вҶ’ еҝ«з…§и®°еҪ• вҶ’ ејӮжӯҘзҙўеј•йҮҚе»ә**гҖӮWrite Guard ж ёеҝғеҠЁдҪңдёә `ADD`гҖҒ`UPDATE`гҖҒ`NOOP`гҖҒ`DELETE`пјӣ`BYPASS` дҪңдёәдёҠеұӮ metadata-only жӣҙж–°еңәжҷҜзҡ„жөҒзЁӢж Үи®°пјҢж•ҙдҪ“й“ҫи·ҜжҜҸдёҖжӯҘеқҮеҸҜиҝҪжәҜгҖӮ

### рҹ”Қ з»ҹдёҖжЈҖзҙўеј•ж“Һ

дёүз§ҚжЈҖзҙўжЁЎејҸвҖ”вҖ”`keyword`пјҲе…ій”®иҜҚпјүгҖҒ`semantic`пјҲиҜӯд№үпјүгҖҒ`hybrid`пјҲж··еҗҲпјүвҖ”вҖ”ж”ҜжҢҒиҮӘеҠЁйҷҚзә§гҖӮеҪ“еӨ–йғЁ Embedding жңҚеҠЎдёҚеҸҜз”Ёж—¶пјҢзі»з»ҹиҮӘеҠЁеӣһйҖҖеҲ°е…ій”®иҜҚжҗңзҙўпјҢе№¶еңЁеҸ‘з”ҹйҷҚзә§ж—¶дәҺе“Қеә”дёӯжҠҘе‘Ҡ `degrade_reasons`гҖӮ

### рҹ§  ж„Ҹеӣҫж„ҹзҹҘжҗңзҙў

жҗңзҙўеј•ж“Һй»ҳи®ӨжҢүеӣӣзұ»ж ёеҝғж„Ҹеӣҫи·Ҝз”ұвҖ”вҖ”**factualпјҲдәӢе®һеһӢпјү**гҖҒ**exploratoryпјҲжҺўзҙўеһӢпјү**гҖҒ**temporalпјҲж—¶й—ҙеһӢпјү**гҖҒ**causalпјҲеӣ жһңеһӢпјү**вҖ”вҖ”е№¶еҢ№й…ҚеҜ№еә”зӯ–з•ҘжЁЎжқҝпјҲ`factual_high_precision`гҖҒ`exploratory_high_recall`гҖҒ`temporal_time_filtered`гҖҒ`causal_wide_pool`пјүпјӣеҪ“ж— жҳҫи‘—дҝЎеҸ·ж—¶й»ҳи®Ө `factual_high_precision`пјҢеҪ“дҝЎеҸ·еҶІзӘҒжҲ–дҪҺдҝЎеҸ·ж··еҗҲж—¶еӣһйҖҖдёә `unknown`пјҲжЁЎжқҝ `default`пјүгҖӮ

### вҷ»пёҸ и®°еҝҶжІ»зҗҶеҫӘзҺҜ

и®°еҝҶжҳҜжңүз”ҹе‘ҪеҠӣзҡ„е®һдҪ“пјҢжӢҘжңүйҡҸж—¶й—ҙиЎ°еҮҸзҡ„ **жҙ»еҠӣеҖјпјҲvitality scoreпјү**гҖӮжІ»зҗҶеҫӘзҺҜж¶өзӣ–пјҡе®ЎжҹҘдёҺеӣһж»ҡгҖҒеӯӨе„ҝжё…зҗҶгҖҒжҙ»еҠӣиЎ°еҮҸгҖҒзқЎзң ж•ҙеҗҲпјҲиҮӘеҠЁзўҺзүҮжё…зҗҶпјүгҖӮ

### рҹҢҗ еӨҡе®ўжҲ·з«Ҝ MCP йӣҶжҲҗ

дёҖеҘ—еҚҸи®®пјҢеӨҡз«ҜжҺҘе…Ҙпјҡ**Codex / Claude Code / Gemini CLI / Cursor / Antigravity**вҖ”вҖ”е…ЁйғЁйҖҡиҝҮеҗҢдёҖеҘ— 9 дёӘ MCP е·Ҙе…· + Skills зӯ–з•ҘеұӮиҝһжҺҘгҖӮ

### рҹ“Ұ зҒөжҙ»йғЁзҪІ

еӣӣз§ҚйғЁзҪІжЎЈдҪҚпјҲA/B/C/DпјүпјҢд»ҺзәҜжң¬ең°еҲ°дә‘з«ҜиҝһжҺҘпјҢж”ҜжҢҒ Docker йғЁзҪІе’ҢдёҖй”®и„ҡжң¬пјҢиҰҶзӣ– macOSгҖҒWindowsгҖҒLinuxгҖӮ

### рҹ“Ҡ еҶ…зҪ®еҸҜи§ӮжөӢжҖ§д»ӘиЎЁзӣҳ

еҹәдәҺ React зҡ„еӣӣи§Ҷеӣҫд»ӘиЎЁзӣҳпјҡ**и®°еҝҶжөҸи§ҲеҷЁ**гҖҒ**е®ЎжҹҘдёҺеӣһж»ҡ**гҖҒ**з»ҙжҠӨз®ЎзҗҶ**гҖҒ**еҸҜи§ӮжөӢжҖ§зӣ‘жҺ§**гҖӮ

| Markdown | react-markdown + remark-gfm | вҖ” | жёІжҹ“и®°еҝҶеҶ…е®№пјҢж”ҜжҢҒ GitHub йЈҺж ј Markdown |
| еӣҫж Ү | [Lucide React](https://lucide.dev/) | вҖ” | з»ҹдёҖеӣҫж ҮдҪ“зі» |

### еҗ„еұӮе®һзҺ°иҜҰи§Ј

#### еҶҷе…ҘжөҒж°ҙзәҝпјҲ`mcp_server.py` вҶ’ `runtime_state.py` вҶ’ `sqlite_client.py`пјү

1. **Write GuardпјҲеҶҷе…Ҙе®ҲеҚ«пјү** вҖ” жҜҸж¬Ў `create_memory` / `update_memory` и°ғз”ЁйғҪе…Ҳз»ҸиҝҮ Write GuardпјҲ`sqlite_client.py`пјүгҖӮеңЁи§„еҲҷжЁЎејҸдёӢпјҢе®ҲеҚ«д»Ҙ **иҜӯд№үеҢ№й…Қ вҶ’ е…ій”®иҜҚеҢ№й…Қ вҶ’ LLMпјҲеҸҜйҖүпјү** зҡ„йЎәеәҸеҲӨе®ҡж ёеҝғеҠЁдҪң `ADD`гҖҒ`UPDATE`гҖҒ`NOOP`гҖҒ`DELETE`пјӣ`BYPASS` з”ұдёҠеұӮжөҒзЁӢеңЁ metadata-only жӣҙж–°еңәжҷҜж ҮжіЁгҖӮеҪ“и®ҫзҪ® `WRITE_GUARD_LLM_ENABLED=true` ж—¶пјҢеҸҜйҖү LLM йҖҡиҝҮ OpenAI е…је®№ API еҸӮдёҺеҶізӯ–гҖӮ

2. **SnapshotпјҲеҝ«з…§пјү** вҖ” еңЁд»»дҪ•дҝ®ж”№еүҚпјҢзі»з»ҹйҖҡиҝҮ `mcp_server.py` дёӯзҡ„ `_snapshot_memory_content()` е’Ң `_snapshot_path_meta()` еҲӣе»әеҪ“еүҚи®°еҝҶзҠ¶жҖҒзҡ„еҝ«з…§гҖӮиҝҷдҪҝеҫ—е®ЎжҹҘд»ӘиЎЁзӣҳдёӯзҡ„е·®ејӮеҜ№жҜ”е’ҢдёҖй”®еӣһж»ҡжҲҗдёәеҸҜиғҪгҖӮ

3. **Write LaneпјҲеҶҷе…ҘиҪҰйҒ“пјү** вҖ” еҶҷе…Ҙиҝӣе…ҘеәҸеҲ—еҢ–йҳҹеҲ—пјҲ`runtime_state.py` вҶ’ `WriteLanes`пјүпјҢеҸҜй…ҚзҪ®е№¶еҸ‘еәҰпјҲ`RUNTIME_WRITE_GLOBAL_CONCURRENCY`пјүгҖӮиҝҷйҳІжӯўдәҶеҚ• SQLite ж–Үд»¶дёҠзҡ„з«һжҖҒжқЎд»¶гҖӮ

4. **Index WorkerпјҲзҙўеј•е·ҘдҪңиҖ…пјү** вҖ” жҜҸж¬ЎеҶҷе…Ҙе®ҢжҲҗеҗҺпјҢејӮжӯҘд»»еҠЎе…ҘйҳҹиҝӣиЎҢзҙўеј•йҮҚе»әпјҲ`runtime_state.py` дёӯзҡ„ `IndexWorker`пјүгҖӮе·ҘдҪңиҖ…жҢү FIFO йЎәеәҸеӨ„зҗҶзҙўеј•жӣҙж–°пјҢдёҚйҳ»еЎһеҶҷе…Ҙи·Ҝеҫ„гҖӮ

#### жЈҖзҙўжөҒж°ҙзәҝпјҲ`sqlite_client.py`пјү

1. **жҹҘиҜўйў„еӨ„зҗҶ** вҖ” `preprocess_query()` еҜ№жҗңзҙўжҹҘиҜўиҝӣиЎҢи§„иҢғеҢ–е’ҢеҲҶиҜҚгҖӮ
2. **ж„ҸеӣҫеҲҶзұ»** вҖ” `classify_intent()` дҪҝз”Ёе…ій”®иҜҚиҜ„еҲҶж–№жі•пјҲ`keyword_scoring_v2`пјүеҲӨе®ҡж„Ҹеӣҫпјҡй»ҳи®Өдёә `factual`гҖҒ`exploratory`гҖҒ`temporal`гҖҒ`causal` еӣӣзұ»пјӣж— жҳҫи‘—е…ій”®иҜҚдҝЎеҸ·ж—¶й»ҳи®Ө `factual`пјҲ`factual_high_precision`пјүпјӣдҝЎеҸ·еҶІзӘҒжҲ–дҪҺдҝЎеҸ·ж··еҗҲж—¶еӣһйҖҖ `unknown`пјҲжЁЎжқҝ `default`пјүгҖӮ
3. **зӯ–з•ҘеҢ№й…Қ** вҖ” ж №жҚ®ж„ҸеӣҫеҢ№й…Қзӯ–з•ҘжЁЎжқҝпјҲеҰӮ `factual_high_precision` дҪҝз”ЁжӣҙдёҘж јзҡ„еҢ№й…Қпјӣ`temporal_time_filtered` ж·»еҠ ж—¶й—ҙиҢғеӣҙзәҰжқҹпјүгҖӮ
4. **еӨҡйҳ¶ж®өжЈҖзҙў** вҖ” жҢүжЎЈдҪҚжү§иЎҢпјҡ
   - **жЎЈдҪҚ A**пјҡзәҜе…ій”®иҜҚеҢ№й…ҚпјҢеҹәдәҺ SQLite FTS
   - **жЎЈдҪҚ B**пјҡе…ій”®иҜҚ + жң¬ең°е“ҲеёҢ Embedding ж··еҗҲиҜ„еҲҶ
   - **жЎЈдҪҚ C/D**пјҡе…ій”®иҜҚ + API Embedding + RerankerпјҲOpenAI е…је®№пјү
5. **з»“жһңз»„иЈ…** вҖ” з»“жһңеҢ…еҗ« `degrade_reasons` еӯ—ж®өпјҢеҪ“д»»дҪ•йҳ¶ж®өеӨұиҙҘж—¶и°ғз”Ёж–№е§Ӣз»ҲдәҶи§ЈжЈҖзҙўиҙЁйҮҸгҖӮ

#### и®°еҝҶжІ»зҗҶпјҲ`sqlite_client.py` вҶ’ `runtime_state.py`пјү

- **жҙ»еҠӣиЎ°еҮҸ** вҖ” жҜҸжқЎи®°еҝҶжңүжҙ»еҠӣеҖјпјҲжңҖеӨ§ `3.0`пјҢеҸҜй…ҚзҪ®пјүгҖӮжҙ»еҠӣжҢүжҢҮж•°иЎ°еҮҸпјҢеҚҠиЎ°жңҹ `VITALITY_DECAY_HALF_LIFE_DAYS=30`гҖӮдҪҺдәҺ `VITALITY_CLEANUP_THRESHOLD=0.35` и¶…иҝҮ `VITALITY_CLEANUP_INACTIVE_DAYS=14` еӨ©зҡ„и®°еҝҶиў«ж Үи®°жё…зҗҶгҖӮ
- **зқЎзң ж•ҙеҗҲ** вҖ” еёҰж•ҙеҗҲеҸӮж•°зҡ„ `rebuild_index` е°ҶзўҺзүҮеҢ–зҡ„е°Ҹи®°еҝҶеҗҲе№¶дёәиҝһиҙҜж‘ҳиҰҒгҖӮ
- **еӯӨе„ҝжё…зҗҶ** вҖ” е®ҡжңҹжү«жҸҸиҜҶеҲ«жІЎжңүжңүж•Ҳи®°еҝҶеј•з”Ёзҡ„и·Ҝеҫ„гҖӮ

| жЎЈдҪҚ | жЈҖзҙўжЁЎејҸ | Embedding | Reranker | йҖӮз”ЁеңәжҷҜ |
|---|---|---|---|---|
| **A** | зәҜ `keyword` | вқҢ е…ій—ӯ | вқҢ е…ій—ӯ | рҹҹў жңҖе°Ҹиө„жәҗпјҢеҲқжӯҘйӘҢиҜҒ |
| **B** | `hybrid` ж··еҗҲ | рҹ“Ұ жң¬ең°е“ҲеёҢ | вқҢ е…ій—ӯ | рҹҹЎ **й»ҳи®Ө**вҖ”вҖ”жң¬ең°ејҖеҸ‘пјҢж— йңҖеӨ–йғЁжңҚеҠЎ |
| **C** | `hybrid` ж··еҗҲ | рҹҢҗ API и°ғз”Ё | вң… ејҖеҗҜ | рҹҹ  жң¬ең°жЁЎеһӢжңҚеҠЎеҷЁпјҲOllama / LM Studioпјү|
| **D** | `hybrid` ж··еҗҲ | рҹҢҗ API и°ғз”Ё | вң… ејҖеҗҜ | рҹ”ҙ иҝңзЁӢ APIпјҢз”ҹдә§зҺҜеўғ |

> **иҜҙжҳҺ**пјҡжЎЈдҪҚ C е’Ң D е…ұдә«зӣёеҗҢзҡ„ж··еҗҲжЈҖзҙўжөҒж°ҙзәҝпјҲ`keyword + semantic + reranker`пјүпјҢеҢәеҲ«д»…еңЁдәҺи·Ҝз”ұеҒҸеҘҪпјҲжң¬ең°дјҳе…Ҳ vs иҝңзЁӢдјҳе…ҲпјүгҖӮ

### рҹ”ј еҚҮзә§еҲ°жЎЈдҪҚ C/D

еңЁ `.env` ж–Үд»¶дёӯй…ҚзҪ®д»ҘдёӢеҸӮж•°гҖӮжүҖжңүз«ҜзӮ№еқҮж”ҜжҢҒ **OpenAI е…је®№ API** ж јејҸпјҢеҢ…жӢ¬жң¬ең°йғЁзҪІзҡ„ Ollama жҲ– LM Studioпјҡ

```bash

RETRIEVAL_EMBEDDING_API_BASE=http://localhost:11434/v1   # дҫӢеҰӮ Ollama
RETRIEVAL_EMBEDDING_API_KEY=your-api-key
RETRIEVAL_EMBEDDING_MODEL=bge-m3

# в”Җв”Җ Reranker жЁЎеһӢ в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
RETRIEVAL_RERANKER_ENABLED=true
RETRIEVAL_RERANKER_API_BASE=http://localhost:11434/v1
RETRIEVAL_RERANKER_API_KEY=your-api-key
RETRIEVAL_RERANKER_MODEL=bge-reranker-v2-m3

# в”Җв”Җ и°ғеҸӮж—Ӣй’®пјҲжҺЁиҚҗ 0.20 ~ 0.40пјүв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җ
RETRIEVAL_RERANKER_WEIGHT=0.25
```

> й…ҚзҪ®иҜӯд№үиҜҙжҳҺпјҡ
> - `RETRIEVAL_EMBEDDING_BACKEND` еҸӘжҺ§еҲ¶ Embedding й“ҫи·ҜгҖӮ
> - Reranker жІЎжңү `RETRIEVAL_RERANKER_BACKEND` ејҖе…іпјҢеҗҜз”ЁдёҺеҗҰз”ұ `RETRIEVAL_RERANKER_ENABLED` жҺ§еҲ¶гҖӮ
> - Reranker иҝһжҺҘеҸӮж•°дјҳе…ҲиҜ»еҸ– `RETRIEVAL_RERANKER_API_BASE/API_KEY/MODEL`пјӣзјәеӨұж—¶жүҚеӣһйҖҖ `ROUTER_*`пјҲе…¶дёӯ base/key иҝҳеҸҜз»§з»ӯеӣһйҖҖ `OPENAI_*`пјүгҖӮ

### еҶҷе…Ҙи·Ҝеҫ„

1. `create_memory` / `update_memory` иҝӣе…Ҙ **Write Lane** йҳҹеҲ—

3. **еҝ«з…§** дёҺзүҲжң¬еҸҳжӣҙи®°еҪ•з”ҹжҲҗ
4. ејӮжӯҘ **Index Worker** е…ҘйҳҹиҝӣиЎҢзҙўеј•жӣҙж–°

### жЈҖзҙўи·Ҝеҫ„

4. иҝ”еӣһ `results` + `degrade_reasons`
